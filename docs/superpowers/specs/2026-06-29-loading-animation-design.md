# Анимация загрузки сайта — дизайн

Дата: 2026-06-29

## Цель

Показывать на старте страницы заставку-оверлей с фирменным знаком на 1 секунду
и блокировать скролл по странице на это время. Заставка должна органично
вписываться в эстетику сайта (стиль Apple, светлая база `#f5f5f7`, фирменный
синий квадрат с белым треугольником).

## Поведение

1. При загрузке поверх всего контента — полноэкранный оверлей цвета `var(--bg)`
   (`#f5f5f7` в светлой теме).
2. По центру — фирменный знак (синий квадрат `.bm-sq` + белый треугольник
   `.bm-tri`), который плавно проявляется и слегка масштабируется/пульсирует.
3. Скролл по странице заблокирован, пока оверлей виден.
4. Оверлей **полностью видим ровно 1 секунду**, затем плавно растворяется
   (`opacity 1→0`, ~0.35s). Скролл разблокируется **в начале** fade-out.
5. После завершения fade узел `#preloader` удаляется из DOM
   (по `transitionend` + страховочный таймаут).

Итоговый тайминг: ~1.0s полной видимости + ~0.35s fade-out до исчезновения.

## Подход (выбран A)

Оверлей живёт в разметке, класс-переключатель ставится на `<html>` через
крошечный инлайн-скрипт в `<head>` — **до первой отрисовки**. Это исключает
проблеск контента (FOUC) и корректно деградирует: при отключённом JS класс не
ставится, оверлей не показывается и скролл не блокируется. Логика 1-секундного
тайминга — в `app.js` (подключается с `defer`).

Отвергнутые альтернативы:
- **B. Вся логика в `app.js`** — из-за `defer` возможен короткий проблеск
  контента до появления оверлея.
- **C. Только CSS** — нельзя надёжно снять блокировку скролла и убрать оверлей
  из DOM ровно через 1с без JS.

## Изменения по файлам

### `index.html`
- В `<head>` — инлайн-скрипт (до `style.css` не обязательно, но до отрисовки):
  `document.documentElement.classList.add('is-loading')`.
- Первым потомком `<body>` — разметка оверлея:
  ```html
  <div id="preloader" aria-hidden="true">
    <svg class="brand-mark" viewBox="0 0 24 24" aria-hidden="true">
      <rect class="bm-sq" width="24" height="24" rx="2"></rect>
      <path class="bm-tri" d="M12 6.8 17.3 16.4 6.7 16.4Z"></path>
    </svg>
  </div>
  ```

### `style.css`
- `#preloader` — `position:fixed; inset:0; display:grid; place-items:center;
  background:var(--bg); z-index:<выше nav и модалок>`.
- Размер знака в оверлее — крупнее навбарного (например, `~64px`).
- Скролл-лок: `html.is-loading{ overflow:hidden }`.
- `@keyframes` появления знака: `transform: scale(.7)→scale(1)` + `opacity 0→1`
  + лёгкий финальный пульс.
- Fade-out: `html.is-loaded #preloader{ opacity:0; pointer-events:none;
  transition: opacity .35s var(--ease) }`.
- `@media (prefers-reduced-motion: reduce)` — без масштаба/пульса, только
  короткий fade; уважаем настройку (в проекте это уже принято для `scroll-behavior`).

### `app.js`
- В конце файла — контроллер:
  ```js
  const html = document.documentElement;
  const pre = document.getElementById('preloader');
  if (pre) {
    setTimeout(() => {
      html.classList.replace('is-loading', 'is-loaded'); // unlock scroll + fade
      const done = () => pre.remove();
      pre.addEventListener('transitionend', done, { once: true });
      setTimeout(done, 600); // страховка, если transitionend не сработает
    }, 1000);
  } else {
    html.classList.remove('is-loading');
  }
  ```

## Граничные случаи
- **JS отключён:** класс `is-loading` не ставится → нет оверлея, нет блокировки.
- **`prefers-reduced-motion`:** без анимаций движения, простой быстрый fade.
- **Тёмные секции:** оверлей использует `var(--bg)` светлой базы (старт страницы
  всегда светлый), цвет совпадает с фоном hero — без визуального шва.

## Критерии готовности
- Заставка видна сразу при загрузке, без проблеска контента.
- Скролл (колесо, тач, клавиатура) не работает, пока оверлей виден.
- Через ~1с оверлей растворяется, скролл возвращается, `#preloader` исчезает из DOM.
- При `prefers-reduced-motion` — без резких движений.
