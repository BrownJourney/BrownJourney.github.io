# Переключение языка (RU / EN) — дизайн

Дата: 2026-06-29
Статус: утверждён, готов к плану реализации

## Цель

Добавить возможность переключения языка интерфейса сайта-портфолио между русским
(текущий) и английским. Переключатель располагается в хедере. Русский остаётся
языком по умолчанию.

## Контекст

Одностраничный статический сайт без сборщика:

- `index.html` (~739 строк) — весь видимый текст inline на русском: nav, мобильное
  меню, hero, услуги, процесс, кейсы (карточки), FAQ, футер, SEO-теги.
- `app.js` (~36 КБ) — ванильный интерактив + массив `CASES` с данными кейсов
  (заголовки, задачи, решения, результаты) на русском. Модал кейса заполняется
  из `CASES` через `openCase(id)` (строки ~556–590).
- `style.css` — стили.

Карточки кейсов в `#casesGrid` — статичный HTML (строки ~400–575). Модальное окно
кейса `#caseModal` заполняется динамически из `CASES`.

## Решения (утверждены)

- **Подход:** data-атрибуты. RU остаётся inline-дефолтом, EN добавляется рядом.
- **Язык по умолчанию:** русский. Выбор сохраняется в `localStorage` и применяется
  при следующих визитах. Автоопределение по браузеру не делаем.
- **Вид переключателя:** одна кнопка-тумблер, показывающая язык, на который
  произойдёт переключение (`🌐 EN`, при активном EN — `🌐 RU`).

## Архитектура

Вся i18n-логика — небольшой модуль в начале `app.js`. Единственная публичная точка
входа — `setLang(lang)`. Никаких новых файлов и зависимостей.

### Хранение переводов

1. **Текст элементов** — атрибут `data-en="English text"` на элементе. Русский
   остаётся как обычный `textContent`. При переключении на EN JS подменяет
   `textContent` на значение `data-en`; при возврате на RU — на сохранённый
   оригинал.

   Чтобы возврат на RU был надёжным, при первом запуске для каждого `[data-en]`
   элемента исходный русский текст сохраняется в `data-ru` (если ещё не задан).

2. **Текст в атрибутах** — конвенция `data-en-<attr>`:
   - `data-en-aria-label` → подменяет `aria-label`
   - `data-en-alt` → подменяет `alt`
   - и аналогично для прочих атрибутов при необходимости.

   Оригинальные значения атрибутов сохраняются в `data-ru-<attr>` при первом запуске.

3. **Документ-уровень** (обрабатываются отдельно, без data-атрибутов):
   - `<html lang>` → `ru` / `en`
   - `<title>`
   - `<meta name="description">`
   - `<meta property="og:title">`, `og:description`, `og:locale` (`ru_RU` / `en_US`)

   Английские значения для них задаются в конфиге i18n-модуля (объект `DOC_META`).

4. **Кейсы (модал)** — в каждый объект `CASES` добавляются параллельные английские
   поля: `title_en`, `summary_en`, `task_en`, `solution_en`, `result_en`,
   `kicker_en`, `role_en`, `ndaSub_en`. Поля карточек (`cardTask`, `cardResult`)
   переводятся через `data-en` в HTML карточек, не через `CASES` (карточки
   статичны). Технические теги (`detailTags`, `cardTags`) не переводятся.

   `openCase(id)` выбирает поле по текущему языку: например
   `getLang() === 'en' ? (d.task_en ?? d.task) : d.task`. Через хелпер
   `t(d, 'task')` для краткости.

### Функция `setLang(lang)`

1. Нормализует `lang` к `'ru'|'en'`, пишет в `localStorage['lang']`.
2. Сохраняет текущий `lang` в модульной переменной (для `openCase`).
3. Проходит по `document.querySelectorAll('[data-en]')` → меняет `textContent`
   (RU из `data-ru`, EN из `data-en`).
4. Проходит по элементам с `data-en-*` атрибутами → меняет соответствующие
   атрибуты.
5. Обновляет документ-уровень: `<html lang>`, `<title>`, meta-теги.
6. Обновляет подпись кнопки-переключателя (`🌐 EN` / `🌐 RU`) и её `aria-label`.
7. Если открыт модал кейса — перезаполняет его текстом на новом языке.

При загрузке (`DOMContentLoaded`/инициализация app.js) вызывается `setLang(initial)`,
где `initial` = сохранённое значение из localStorage или `'ru'`.

### Переключатель в хедере

- HTML: `<button class="lang-toggle" id="langToggle" type="button">`.
  Размещается в `.nav-cta` (рядом с Telegram-кнопкой и бургером). Дублируется в
  `.mobile-menu` для мобильной навигации.
- Содержимое: иконка-глобус (inline SVG или эмодзи 🌐) + подпись целевого языка.
- Стили в `style.css` в духе существующих кнопок (`btn-ghost`-подобный, компактный),
  видим на десктопе и мобильных.
- Обработчик: клик → `setLang(getLang() === 'ru' ? 'en' : 'ru')`. Один обработчик
  обслуживает обе копии (десктоп + мобайл) или вешается на оба элемента.

## Поток данных

```
загрузка → читаем localStorage['lang'] (или 'ru')
         → setLang(initial): снимок RU в data-ru → применяем язык ко всем узлам,
           атрибутам, документ-мете, кнопке
клик по кнопке → setLang(другой язык) → перерисовка + сохранение
открытие кейса → openCase: t(d, field) по текущему языку
```

## Обработка ошибок / краевые случаи

- `localStorage` недоступен (приватный режим) → оборачиваем чтение/запись в
  try/catch, при ошибке работаем в памяти, дефолт `ru`.
- Элемент без `data-en` → остаётся на русском (намеренно, для неизменяемых строк
  вроде имени бренда; чекер ниже помогает не забыть нужные).
- Поле кейса без `*_en` → fallback на русский оригинал (`?? d.field`).
- Переключение при открытом модале → модал перезаполняется, не закрывается.

## Тестирование

Тест-раннера на проекте нет — проверка ручная в браузере плюс вспомогательный
скрипт-чекер.

- **Скрипт-чекер** (одноразовый, для разработки): обходит видимые текстовые узлы в
  `index.html` и сообщает о тех, у кого нет `data-en` — чтобы ничего не пропустить
  при переводе. Не часть рантайма сайта.
- **Ручные сценарии:**
  1. Переключение RU→EN меняет весь видимый текст (nav, hero, услуги, процесс,
     карточки кейсов, FAQ, футер).
  2. Перезагрузка страницы сохраняет выбранный язык.
  3. Модал кейса открывается на текущем языке; переключение при открытом модале
     обновляет его текст.
  4. Мобильное меню и его переключатель работают так же.
  5. `<html lang>`, `<title>`, meta-описание меняются (проверка в DevTools).
  6. Атрибуты `aria-label`/`alt` переключаются.

## Объём работ (для плана)

1. i18n-модуль в `app.js` (`setLang`, `getLang`, `t`, инициализация).
2. Кнопка-переключатель: HTML в nav + мобильное меню, стили в `style.css`,
   обработчик в `app.js`.
3. Простановка `data-en` / `data-en-*` на весь переводимый контент `index.html`
   + английские тексты.
4. Английские поля `*_en` в массиве `CASES` (`app.js`).
5. Документ-мета: английские значения title/description/og в конфиге, применение
   в `setLang`.
6. Скрипт-чекер покрытия `data-en` + ручная проверка сценариев.

## Вне объёма (YAGNI)

- Автоопределение языка по браузеру.
- Третий и последующие языки (структура расширяема, но не реализуется).
- Серверный рендер / отдельные URL на язык / sitemap-hreflang.
- Перевод alt у изображений кейсов в галерее модала (только основные alt).
